# Finitexo Code Matrix v0.4.3 - Frozen Dataset Policy

## Rules

- Frozen tasks must not be modified in place.
- Changes require an explicit future version bump.
- Provenance records must remain linked to every task.
- Human-review promoted candidates must keep authorization records.
- Provider execution is not authorized in v0.4.3.
- Model comparison is not authorized in v0.4.3.
- External superiority claims are not authorized.
- Statistical significance claims are not authorized.
- Dataset existence does not imply provider, model, Xendris, production, or
  statistical validation.

## Authorized Claim

```txt
Finitexo Code Matrix v0.4.3 contains an explicitly versioned expanded frozen external/adapted candidate dataset with n=10 tasks, provenance records, and reproducible hashes.
```

## Blocked Claims

- provider superiority demonstrated;
- Xendris superiority demonstrated;
- external benchmark performance validated;
- statistical significance established;
- production-readiness proven;
- verified third-party external benchmark established.
