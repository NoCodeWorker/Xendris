# Finitexo Code Matrix v0.3 - Interpretation Policy

## Principle

The benchmark exists to test a hypothesis, not to protect it.

## Allowed Language

- "benchmark-local diagnostic result"
- "signal repeated"
- "signal did not repeat"
- "no clear advantage"
- "baseline matched"
- "baseline outperformed"
- "inconclusive under this protocol"
- "blocked for interpretation"

## Forbidden Language

- "universal superiority"
- "general coding superiority"
- "production-ready"
- "provider superiority"
- "model superiority"
- "proved"
- "guaranteed"

## Small Samples

Runs below 20 samples may be useful for budget validation, plumbing, or smoke
testing, but they cannot support performance claims.
