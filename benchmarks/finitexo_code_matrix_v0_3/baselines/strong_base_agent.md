# strong_base_agent

## Purpose

General strong baseline with ordinary engineering discipline.

## Behavior

- Preserve the public API.
- Minimize patch size.
- Address visible and likely hidden edge cases.
- Avoid unrelated files.
- Avoid claiming success without test evidence.
- Avoid unsafe code patterns.

## Interpretation

This baseline is stronger than a direct answer baseline but still lacks the
explicit test-discipline and adversarial framing of the strongest baseline.
