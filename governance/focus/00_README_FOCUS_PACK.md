# Frontera C Focus Pack

Este paquete devuelve el proyecto al eje principal desde el estado actual, sin depender de memoria informal ni de instrucciones manuales sueltas.

## Objetivo

Restaurar y proteger el foco de investigación:

> **Frontera C-Mayor: `c` como membrana causal-informacional entre dominios físicos de realidad, información, coherencia y observabilidad.**

## Problema corregido

El proyecto se desvió desde el núcleo principal hacia un subestudio auxiliar:

```text
heating_power_W → visibilidad/contraste → óptica térmica → cámaras/sensores
```

Ese subestudio puede conservarse, pero queda degradado a:

```text
AUXILIARY_ONLY
```

No debe volver a ocupar el lugar del tronco principal.

## Orden recomendado de instalación en Codex

1. Pegar o añadir al repositorio `01_FOCUS_CONTRACT.md`.
2. Crear o actualizar `02_PROJECT_RAILS.md`.
3. Ejecutar `03_RECENTER_FROM_CURRENT_STATE.md`.
4. Ejecutar `04_ARTIFACT_AUDIT_PROTOCOL.md`.
5. Mantener `05_PERIODIC_AUDIT_TRIGGERS.md` como política de control.
6. Añadir `06_RESPONSE_TEMPLATE.md` como formato obligatorio de salida.
7. Usar `07_DRIFT_RECOVERY_PROMPTS.md` cuando Codex derive.
8. Actualizar siempre `08_PROJECT_STATUS_SCHEMA.md`.
9. Mantener `09_SUBSTUDY_REGISTRY.md` para impedir que ramas auxiliares sustituyan el núcleo.

## Regla central

Ningún artefacto, benchmark, script, experimento o razonamiento puede ser tratado como avance principal si no conecta explícitamente con:

- `c`,
- causalidad,
- conos de luz,
- horizontes,
- información,
- observabilidad,
- coherencia/decoherencia,
- medición,
- falsabilidad de Frontera C-Mayor.
