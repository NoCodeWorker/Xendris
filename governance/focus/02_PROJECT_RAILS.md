# Project Rails — Frontera C-Mayor

## Purpose

This file defines the rails that keep the project from drifting into adjacent but non-primary studies.

## Rail 1 — The Primary Hypothesis

Frontera C-Mayor proposes:

```text
c is not only a maximum propagation speed.
c may also define a causal-informational membrane:
a boundary between physical domains that can and cannot exchange coherent, causal, measurable information with an observer.
```

## Rail 2 — The Central Question

The central question is:

```text
Does the speed of light act only as a velocity limit, or also as a structural membrane organizing physical accessibility, observability, coherence, and information transfer?
```

## Rail 3 — The Conceptual Stack

All work must map to this stack:

```text
Level 1 — c
Level 2 — causal structure
Level 3 — information transfer
Level 4 — observer and measurement
Level 5 — coherence/decoherence
Level 6 — inaccessible domains and horizons
Level 7 — falsifiable consequences
Level 8 — auxiliary models
```

Anything that starts at Level 8 cannot be promoted to Level 1 without a clear argument.

## Rail 4 — Allowed Core Topics

Allowed CORE topics:

- speed of light,
- relativistic causality,
- Minkowski spacetime,
- light cones,
- timelike/lightlike/spacelike separation,
- event horizons,
- cosmological horizons,
- causal diamonds,
- observer-dependent reality access,
- information transfer limits,
- measurement,
- decoherence,
- boundary between observable and non-observable physical domains.

## Rail 5 — Allowed Bridge Topics

Allowed BRIDGE topics:

- entropy,
- information loss,
- recoverability,
- signal coherence,
- phase relation,
- quantum/classical transition,
- causal inference,
- observability,
- measurement limits,
- horizon thermodynamics,
- holographic principle,
- black-hole information problem.

Bridge topics must explicitly connect back to `c`.

## Rail 6 — Auxiliary Topics

These are allowed only as secondary work:

- `heating_power_W`,
- visibility,
- contrast,
- optical degradation,
- thermal turbulence,
- image distortion,
- cameras,
- sensors,
- LiDAR,
- signal restoration,
- benchmark datasets.

Auxiliary topics must be tagged:

```yaml
classification: AUXILIARY
primary_goal_relevance: indirect
may_validate_main_hypothesis: false
```

## Rail 7 — Forbidden Substitutions

Do not substitute:

```text
contrast for causal accessibility
visibility for observability
heat for c
camera signal for physical reality
image restoration for Frontera C validation
benchmark readiness for theory validation
```

## Rail 8 — Progress Accounting

Progress must be accounted separately:

```yaml
CORE_PROGRESS:
BRIDGE_PROGRESS:
AUXILIARY_PROGRESS:
OFF_TRACK_DEBT:
```

Auxiliary progress cannot increase the validation score of Frontera C-Mayor unless a formal bridge is written.

## Rail 9 — Formal Bridge Requirement

Any auxiliary study must include a bridge note:

```markdown
## Bridge to Frontera C-Mayor

- What core concept does this auxiliary study model?
- Which part of the causal-informational membrane does it illuminate?
- What does it not prove?
- Why is this not being treated as validation?
- What would be required to promote this from AUXILIARY to BRIDGE?
```

## Rail 10 — Stop Condition

If Codex cannot state how a task connects to `c`, causal structure, information transfer, coherence, observability, or falsability, it must stop and ask for recentering rather than continue.
