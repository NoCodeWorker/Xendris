# Phygn Roadmap — v0.3 to v0.4

## Estado v0.3

Phygn v0.3 debe demostrar que el marco puede operar en software.

Objetivos v0.3:

```txt
backend científico
tests
API
firma Q/B
escala L justificada
huella epistemológica en canal cuántico
cota negativa mesoscópica
Claim Gatekeeper
```

## Hito 1 — Backend estable

Criterios:

```txt
pytest pasa completo
API arranca
README actualizado
endpoints funcionales
```

## Hito 2 — Primer reporte automático

Generar un reporte Markdown desde el caso mesoscópico:

```txt
inputs
fórmulas
resultados
trace_type
claims permitidos
claims bloqueados
limitaciones
```

Archivo sugerido:

```txt
reports/mesoscopic_negative_bound_report.md
```

## Hito 3 — Export para paper

Crear exportadores:

```txt
JSON
Markdown
LaTeX snippet
```

Para usar resultados en el paper.

## Hito 4 — Frontend Cockpit

Solo tras backend estable.

Pantallas:

```txt
Dashboard
Signature Lab
Operational Scale Review
Epistemic Trace Lab
Predictive Gain
Claim Gatekeeper
Case Studies
```

## Hito 5 — Caso físico más fuerte

Elegir uno:

```txt
A. Interferómetro mesoscópico con decoherencia base
B. Black hole information toy model
C. N-qubit noisy channel
D. Bound on accessible information under causal constraints
```

## Hito 6 — Predictive Gain real o cota negativa no trivial

Frontera C/Phygn empieza a ganar fuerza si produce:

```txt
Gain_C > 0
```

o una:

```txt
NEGATIVE_BOUND_TRACE
```

que descarte familias de claims o regímenes de búsqueda.
