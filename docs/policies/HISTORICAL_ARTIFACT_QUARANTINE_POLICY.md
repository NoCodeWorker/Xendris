# Historical Artifact Quarantine Policy

## Purpose

This policy separates active benchmark evidence from historical rejected
benchmark artifacts. It prevents old rejected summaries from blocking release
merely by existing, while preserving traceability and preventing their use as
evidence.

## Historical Rejected Artifact

A historical rejected artifact is a benchmark summary, result file, or report
that:

- was generated by an older runner or incomplete metadata path;
- is rejected by `runs/benchmark_evidence_registry.json`;
- is retained for audit history;
- must not be used as admitted benchmark evidence.

## Location

Historical rejected artifacts may remain in their original paths when moving
them would break traceability. Their status must be recorded in:

```txt
docs/benchmarks/HISTORICAL_REJECTED_ARTIFACTS.md
```

Future rejected scratch outputs should be moved to an ignored scratch location
or regenerated through the canonical runner before being considered for
evidence admission.

## Reference Rules

Historical rejected artifacts may be referenced only when the surrounding text
clearly marks them as:

- rejected;
- blocked;
- historical;
- not admitted;
- superseded;
- not valid for public claims.

They must not be cited as active benchmark evidence, model quality evidence,
roadmap support, or public comparison support.

## Evidence Exclusion

Quarantined historical rejected artifacts remain visible in suite audit and
evidence registry outputs, but they are excluded from active release blockers
when they are listed in the quarantine manifest.

They still remain rejected evidence.

## Traceability

Traceability is preserved by keeping:

- original artifact path;
- rejection reason;
- supersession status;
- canonical admitted replacement when one exists;
- allowed reference status.

## Canonical Replacement Rule

When a canonical admitted replacement exists, public docs should cite the
replacement rather than the historical rejected artifact.

For Trust Traps v0.1, the canonical admitted dry-run summary is:

```txt
runs/trust_excellence_check/deepseek_vs_xendris_trust_traps_v0_1_summary.json
```

## Enforcement

The release flow checks:

```powershell
.\.venv\Scripts\python.exe scripts\check_benchmark_evidence_citations.py
.\.venv\Scripts\python.exe scripts\audit_benchmark_suite_excellence.py --fail-on-blockers
.\.venv\Scripts\python.exe scripts\release_gate_v0_2_2.py
```

Unsafe citations still fail. Quarantined rejected historical artifacts do not
block release solely because they exist.
