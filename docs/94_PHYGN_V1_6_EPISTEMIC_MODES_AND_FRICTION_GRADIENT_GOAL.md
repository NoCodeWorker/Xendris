# Phygn v1.6 — Epistemic Modes & Friction Gradient Goal

## 0. Purpose

Phygn has become increasingly rigorous:

```txt
source audit
claim gatekeeper
baseline comparison
candidate failure modes
positive prediction gate
kill/pivot criteria
synthetic benchmark
```

This rigor is necessary.

But there is a serious product and scientific risk:

```txt
If Phygn blocks 99.99% of early ideas, it becomes a creativity killer.
```

A system that treats a dream, a sketch, a hypothesis, a publication claim and a real-money decision the same way will paralyze human exploration.

v1.6 fixes this.

---

## 1. Core principle

```txt
Do not block ideas.
Block premature authority.
```

Or:

```txt
Hypotheses may breathe.
Claims must bleed.
Actions must be gated.
```

---

## 2. The problem

Human cognition needs:

```txt
intuition
pálpito
analogy
creative error
play
aesthetic attraction
speculative exploration
```

If Phygn blocks these at birth, it becomes:

```txt
the Grinch of scientific imagination
```

But if Phygn lets intuition become:

```txt
public truth
financial advice
clinical recommendation
scientific validation
automated execution
```

without evidence, it becomes dangerous.

Therefore, Phygn needs modes.

---

## 3. v1.6 goal

Implement:

```txt
Epistemic Modes
Dream-to-Claim Ladder
Friction Gradient
Risk-Weighted Gatekeeping
Hypothesis Incubation
Action Permission Matrix
```

---

## 4. Epistemic modes

```txt
DREAM_MODE
EXPLORATION_MODE
HYPOTHESIS_MODE
TEST_DESIGN_MODE
CLAIM_MODE
PUBLICATION_MODE
FINANCIAL_ACTION_MODE
AUTOMATED_EXECUTION_MODE
```

Each mode has its own gate.

---

## 5. Friction Gradient

The level of evidence required depends on risk:

```txt
low risk  -> low friction
high risk -> high friction
```

Examples:

```txt
private intuition          -> almost no blocking
internal hypothesis        -> structure required
testable hypothesis        -> observable/failure required
paper claim                -> source/benchmark required
financial recommendation   -> source/risk/backtest required
automated real-money trade -> maximum gate
```

---

## 6. New distinction

Phygn must separate:

```txt
idea permission
hypothesis permission
claim permission
action permission
execution permission
```

An idea may be allowed while the claim is blocked.

Example:

```txt
Idea:
"Maybe Frontera C modulates decoherence."

Status:
IDEA_ALLOWED
HYPOTHESIS_REQUIRES_FORMALIZATION
CLAIM_BLOCKED
ACTION_NOT_APPLICABLE
```

This prevents Phygn from killing creativity while preserving rigor.

## 8. What v1.6 may unlock

Allowed:

```txt
more ideas may enter Phygn
hypotheses can be incubated without being killed
claims remain gated
actions remain risk-weighted
```

Not allowed:

```txt
lowering rigor for public claims
lowering rigor for financial execution
calling intuition evidence
allowing unsupported claims as truth
```
