## 2. What Must Be Falsifiable

The claim to test is not:

```txt
c is a speed limit.
```

That is established physics.

The claim to test is:

```txt
c can be modeled as a causal-informational membrane that organizes physical accessibility, information transfer, coherence, recoverability, and observer-relative measurability in a way not reducible to existing formulations.
```

## 3. Falsification Routes

### Route A — Redundancy

Frontera C-Mayor is falsified as a novel framework if every proposed result is fully reducible to:

- standard light-cone causality,
- ordinary horizon causal structure,
- standard decoherence,
- standard information theory,
- ordinary measurement limits.

Outcome:

```txt
FALSIFIED_AS_NOVEL
```

### Route B — Incoherence

The framework fails if its definitions are inconsistent, circular, or unable to distinguish:

- causal access,
- information access,
- measurement access,
- coherence,
- recoverability.

Outcome:

```txt
FALSIFIED_AS_FORMAL_MODEL
```

### Route C — Conflict With Established Physics

The framework fails or requires radical revision if it implies:

- superluminal causal signalling,
- violation of Lorentz invariance without evidence,
- contradiction with standard horizon causal ordering,
- unsupported modification of quantum measurement rules.

Outcome:

```txt
REJECT_OR_REFORMULATE
```

### Route D — No Failure Condition

If every observation can be reinterpreted as compatible, the hypothesis is not scientific in its current form.

Outcome:

```txt
BLOCKED_AS_INFALSIFIABLE
```

### Route E — Auxiliary Substitution

If an auxiliary benchmark is treated as proof of the core membrane, the validation claim is invalid.

Outcome:

```txt
CLAIM_REJECTED_AUXILIARY_SUBSTITUTION
```

## 4. Possible Positive Pressure

Positive pressure would require a result that:

1. states a core claim involving `c`,
2. states the known-physics baseline,
3. defines what the membrane model predicts differently,
4. can fail,
5. is not merely an analogy,
6. is reproducible as mathematics, theorem, or empirical result.

## 5. What Counts Only as Auxiliary Support

The following can support analogies or methods but cannot falsify or validate the core directly:

- visibility/contrast degradation,
- thermal optical effects,
- image or sensor signal corruption,
- local benchmarks,
- PredictiveGain on non-core datasets,
- heating-power axis expansion.
