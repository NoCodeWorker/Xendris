# Frontera C v0.3 — Manifiesto operacional

## Cambio estratégico

Frontera C v0.2.2 queda clasificada como una arquitectura metodológica físico-informacional con autocontrol epistémico. Su valor actual no reside en haber descubierto una nueva ley física, sino en haber construido un sistema formal para:

- distinguir definiciones, lemas, hipótesis, modelos y extensiones;
- bloquear claims infalsables;
- exigir huella epistemológica;
- separar núcleo físico, núcleo onto-epistémico y extensiones especulativas;
- convertir la frontera causal articulada por c en una arquitectura operacional.

A partir de v0.3, el objetivo ya no es ampliar la teoría general, sino producir el primer caso de estudio calculable.

## Decisión central

```txt
DEC-014
La generalidad puede esperar. La predicción no.
```

```txt
DEC-015
Toda entidad formal nueva deberá estar vinculada a una función computacional, benchmark, cota, comparación con modelo base o cálculo operativo.
```

```txt
DEC-016
Las extensiones cognitivas quedan fuera del paper principal y se trasladan a un documento separado.
```

```txt
DEC-017
El criterio de progreso de Frontera C v0.3 será la producción de una huella, una cota negativa, un benchmark no trivial o un Predictive Gain calculable.
```

## Alcance de v0.3

Frontera C v0.3 debe centrarse en tres piezas:

1. Principio de selección no arbitraria de escala operacional L.
2. Caso de estudio calculable de huella epistemológica.
3. Caso de estudio físico con cota negativa usando Q,B.

## Claim central revisado

> Frontera C no pretende reemplazar teorías físicas establecidas. Propone una arquitectura operacional para clasificar fronteras físicas, controlar claims y buscar huellas predictivas en sistemas concretos.

## Métrica de éxito

Un caso v0.3 es exitoso si produce al menos uno de estos resultados:

```txt
STRUCTURAL_TRACE verificable
NEGATIVE_BOUND_TRACE útil
PREDICTIVE_TRACE simulada
comparación explícita con modelo base
Gain_C calculado aunque sea cero
claim bloqueado correctamente por el Gatekeeper
```
