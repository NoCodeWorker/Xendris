# Phygn v0.7 — Benchmark Data Protocol

## 0. Propósito

Este documento define cuándo Phygn puede usar datos o curvas de referencia como `y_true`.

v0.6 bloqueó correctamente:

```txt
Gain_C = undefined without y_true
```

v0.7 debe definir qué cuenta como `y_true`.

## 1. Problema

No todo vector de datos es benchmark.

Un vector puede ser:

```txt
placeholder
synthetic
simulated
literature extracted
experimental
```

Cada uno permite claims distintos.

## 2. BenchmarkDataset

```python
class BenchmarkDataset(BaseModel):
    dataset_id: str
    name: str
    observable: str
    t: list[float]
    y_true: list[float]
    provenance_type: str
    source_ids: list[str]
    generation_method: str | None
    parameters: dict[str, float]
    uncertainty: list[float] | None
    epsilon_exp: float | None
    allowed_uses: list[str]
    forbidden_uses: list[str]
```

## 3. Provenance types

```txt
PLACEHOLDER:
UI/dev only. Cannot compute Gain_C.

SYNTHETIC:
generated from explicit formula. Can compute synthetic gain only.

SIMULATED:
generated by documented simulation. Can compute simulated gain.

LITERATURE_EXTRACTED:
extracted from paper/table/figure. Requires source and extraction notes.

EXPERIMENTAL:
real experimental data. Requires source, uncertainty and context.
```

## 4. Gain permissions

```txt
PLACEHOLDER:
Gain_C forbidden.

SYNTHETIC:
SyntheticGain allowed.
PredictiveGain forbidden.

SIMULATED:
SimulatedGain allowed.
Physical claim limited.

LITERATURE_EXTRACTED:
PredictiveGain candidate if extraction is valid.

EXPERIMENTAL:
PredictiveGain allowed if source/provenance/uncertainty valid.
```

## 5. BenchmarkReadiness

```python
class BenchmarkReadinessResult(BaseModel):
    dataset_id: str
    readiness_status: str
    can_compute_gain: bool
    gain_label: str | None
    allowed_claim_level: int
    blocked_reason: str | None
    required_actions: list[str]
```

Statuses:

```txt
NOT_A_BENCHMARK
SYNTHETIC_READY
SIMULATED_READY
LITERATURE_READY_WITH_LIMITATIONS
EXPERIMENTAL_READY
REJECTED
```

## 6. Synthetic benchmark protocol

A synthetic benchmark may be useful to test code.

It must include:

```txt
formula
parameters
seed if random
purpose
forbidden physical interpretation
```

Allowed claim:

```txt
The engine computes gain correctly on synthetic benchmark data.
```

Blocked claim:

```txt
Phygn has predictive physical gain.
```

## 7. Literature-extracted protocol

Requires:

```txt
source_id
figure/table reference
extraction method
uncertainty estimate if available
digitization note if applicable
```

If missing:

```txt
LITERATURE_REQUIRES_EXTRACTION_NOTES
```

## 8. Experimental protocol

Requires:

```txt
source_id
experimental system
observable
uncertainty
conditions
units
preprocessing
```

If missing uncertainty:

```txt
EXPERIMENTAL_REQUIRES_UNCERTAINTY
```

## 9. Error metric compatibility

Before computing error:

```txt
same length
same time grid or interpolation protocol
finite values
same observable
same units
```

If not:

```txt
BENCHMARK_INCOMPATIBLE
```
